GFW 通过不同手段对国外网站造成 DDoS 攻击事件记录

  1. Craig Hockenberry 的服务器遭遇了莫名其妙的 DDoS

  2. 2015-01-22 Craig Hockenberry 发表「Fear China」

    GFW 通过 DNS 污染让对 facebook.com 等网站的访问指向了一个随机的 IP，而这个 IP 恰好指向 Craig Hockenberry 的服务器，大量的网络请求使这台服务器宕机。

    http://furbo.org/2015/01/22/fear-china/

  3. 2015-01-28 Craig Hockenberry 发表「Grass Mud Horse」

    带着好奇和愤怒 Craig 调查了下什么是 GFW，然后他用 Grass Mud Horse 来代表自己的心情和文章的标题…

    http://furbo.org/2015/01/28/grass-mud-horse/

  4. GitHub 遭到大规模 DDoS 攻击

  5. 2015-03-28 GitHub 发表「Large Scale DDoS Attack on github.com」

    Github 在 blog 上表示正遭遇大规模 DDoS 攻击

     

    > The attack began around 2AM UTC on Thursday, March 26

     

    但除了开始时间，GitHub 并没有说明有关攻击的其他信息。

     

    HN link: https://news.ycombinator.com/item?id=9284226

    https://github.com/blog/1981-large-scale-ddos-attack-on-github-com

  6. 2015-03-28 很多网友在翻墙访问国内网站时看到「WARNING: malicious javascript detected on this domain」提示

    http://www.v2ex.com/t/179698

  7. 2015-03-29 Solidot 发表「对GitHub的大规模DDoS攻击已超过80个小时」

    简要报道了 GFW 攻击的方式和过程，其中包括最具有「创造性」也是最下作的

     

    “劫持百度 JS 文件利用中国海外用户的浏览器每 2 秒向托管在 GitHub 上的两个反审查项目发出请求”

     

    这一方式

     

    两个反审查项目分别是

    https://github.com/greatfire/

    和

    https://github.com/cn-nytimes/

    http://www.solidot.org/story?sid=43506

  8. 2015-03-31 「GitHub 受到 GFW 攻击是 Greatfire 将镜像放到 GitHub 所引发」这一事实引起社区热议

    争议焦点由 GFW 转移到 Greatfire，关键词：自我审查，民主和正义

    https://www.v2ex.com/t/180150

  9. 2015-03-31 Greatfire 发表「中国当局的使用数百万网民进行网络攻击」

    https://zh.greatfire.org/blog/2015/mar/chinese-authorities-compromise-millions-cyberattacks

  10. 2015-03-31 NETRESEC 发表「China's Man-on-the-Side Attack on GitHub」

    权威性地揭露了攻击的技术细节

     

    HN link: https://news.ycombinator.com/item?id=9293849

    http://www.netresec.com/?page=Blog&month=2015-03&post=China%27s-Man-on-the-Side-Attack-on-GitHub

  11. 2015-04-24 Google 发表「A Javascript-based DDoS Attack as seen by Safe Browsing」

    Google 站在观察者角度客观说明了一些攻击相关的技术和数据

    http://googleonlinesecurity.blogspot.jp/2015/04/a-javascript-based-ddos-attack-as-seen.html
